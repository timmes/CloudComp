Getting Started

System Requirements

  • Modern web browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
  • JavaScript enabled
  • Local storage enabled (for data persistence)
  • No server or installation required

First Time Setup

  1. Open dist/index.html in your browser — this is the single self-contained production build
  2. The application initialises with empty data
  3. Start by importing learning data or by creating teams and campaigns manually
  4. Your data is saved automatically to the browser's local storage

Tip: Bookmark the file path for quick access. Everything persists between sessions until you clear the browser data.

Developer Setup

If you cloned the repository and want to work on the app, install dependencies and run the dev server:

  • npm install — install dependencies
  • npm run dev — start the Vite dev server with hot reload
  • npm run build — rebuild dist/index.html
  • npm test — run the Vitest suite

Light & Dark Mode

The header has a sun/moon icon next to the Docs button. Click it to switch themes. Your choice is remembered, and on first visit the app follows your OS theme preference.

Activity Overview

Two side-by-side widgets summarise activity composition:

  • Course Types — the top five course types by activity count, with a horizontal bar showing share. Click a row to filter the dashboard by that course type; click again (or the × on the pill at the top) to clear.
  • Completion Rate — the share of activities with status completed versus in-progress/enrolled. When a course-type filter is active, the rate is scoped to that type and the label updates accordingly.

Leaderboards & Sub-tabs

Below the metrics, three sub-tabs swap the main panel:

  • Users — top 10 individuals for the selected time period.
  • Teams — top 10 teams for the selected time period (only shown when at least one team exists).
  • Campaigns — a campaign selector plus per-campaign metrics (participants, active participants, eligible activities, points awarded) and a campaign leaderboard. Only shown when at least one campaign exists.

Time Period Filter

The Month / Quarter / Year / Overall toggle on the right of the sub-tabs scopes the user and team leaderboards (and the Recent Activities feed). Overall shows all-time totals. The toggle is hidden on the Campaigns sub-tab, where the time window is determined by each campaign's start/end dates.

Recent Activities

Shows the 10 most recent completed activities within the selected time period, with user name, activity title, date, and points earned. Hidden on the Campaigns sub-tab.

Importing Data

Course Activity Files

Upload Excel (.xlsx) or CSV files with course completion data. The importer auto-detects the format (standard course export vs. classroom training).

ColumnRequiredDescription
EmailYesUser's email address (stored lowercased)
Course IDYesUnique course identifier (used for dedup)
Course TitleYesName of the course
StatusYesCOMPLETED, IN PROGRESS, or ENROLLED
LevelNofoundational / associate / professional / specialty
CourseTypeNoFree-text type label (e.g. Skill Builder Course, Cloud Quest Role) — mapped to a point category
Completed onYes*Completion date — required only for completed courses
ScoreNoQuiz score (0–100)

Classroom Training (ILT) Files

ILT files are auto-detected: Name, Email, Status columns without a Course ID.

ColumnDescription
NameAttendee's full name
EmailAttendee's email address
StatusCompleted / Not Completed

Classroom training awards 100 points per session by default.

Teams Meeting Files

CSV exports from Microsoft Teams. The parser extracts meeting titles and attendee emails. Each attendee receives the Live Webinar point value (25 by default).

Import Process

  1. Go to the Import tab
  2. Select course activity files and/or Teams meeting files
  3. Click Process All Files — a spinner appears next to the Processing Log title while work is in flight
  4. Review the Processing Log for any warnings or skipped rows
  5. Check the dashboard for updated statistics

Duplicates: the importer skips activities that match an existing one on userId + externalId (case-insensitive). An incoming completed row will upgrade an existing in_progress/enrolled activity instead of being skipped.

Excel dates: .xlsx files store dates as serial numbers; the importer converts them via cellDates: true so they parse correctly. If you imported an .xlsx file with an older build and see — (or 1/1/1970) in the Last Activity column, re-import the file with this version to fix the dates.

Managing Users

Users are created automatically when importing activity data, or via the Add Points dialog. Each user is identified by their lowercase email.

Bulk Operations

  1. Select users via the row checkboxes (or the header checkbox to select the page)
  2. Choose an action from the toolbar that appears: Award Points, Assign to Team, or Export Selected (downloads JSON)
  3. Fill in the details and confirm

Manual Points

Click Add Points to award custom points for special achievements. Start typing a name or email to auto-suggest existing users; if no match exists, the email is used as-is and a new user is created.

Managing Teams

Creating Teams

  1. Navigate to the Teams tab
  2. Click Create Team
  3. Enter name, description, and choose a color

Managing Members

Click Manage on any team to add or remove members. Team points are automatically derived from member activity totals.

Tip: A user can belong to one team at a time. Assigning to a new team automatically removes them from the previous one.

Managing Activities

The Activities tab lists every imported or manually-added activity with its user, type, level, points, status, completion date, and source.

Filtering & Sorting

Use the search box to filter by any text in a row. The drop-down narrows further by Current Month, Courses, or Events. Click the sort buttons to order by points or completion date.

Bulk Operations

  • Adjust Points — add, subtract, multiply, or set values across selected activities; the adjustment history is preserved on each activity.
  • Delete Selected — remove activities (team and user totals are recalculated).
  • Export Selected — download selected activities as JSON.

Point Configuration

The Configuration tab lets you edit the points awarded for each activity type. Values are organised into five sections matching the categories above:

  • Self-Paced Digital Training
  • Live Learning Sessions
  • Certifications & Exams
  • Gamified Events & Challenges
  • Community Engagement

Note: Configuration changes apply to future imports and future manual awards. Existing activities keep the points they were originally awarded.

Saving Changes

Edit the numbers inline and click Save Changes in the top right. A short confirmation appears next to the button.

Reset All Data

The danger zone card at the bottom of the Configuration tab contains a Reset button that permanently deletes all users, teams, activities, campaigns, and configuration. A confirmation modal requires you to type the confirmation phrase before the action runs.

Campaigns

Campaigns let you track learning progress within a specific time window for a defined set of participants.

Creating a Campaign

  1. Go to the Campaigns tab
  2. Click Create Campaign
  3. Set name, dates, and link teams or individual participants
  4. Save as Draft, then Activate when ready

Participant Resolution

Linking a team automatically includes all its members as campaign participants. Individual users can also be added directly. The union of both is deduplicated, so a user who is both directly listed and in a linked team only counts once.

Leaderboards

Each campaign has its own user and team leaderboards on the Campaigns tab detail view, scoped to activities completed within the campaign's date window by eligible participants.

The dashboard also has a Campaigns sub-tab: pick a campaign from the dropdown to see its participant count, eligible activities, points awarded, and the per-campaign user leaderboard at a glance. (For the team leaderboard within a campaign, open the campaign detail view in the Campaigns tab.)

Status Lifecycle

StatusDescription
draftBeing configured, not yet tracking
activeTracking activities within the date window
completedEnd date passed, results finalised
archivedHidden from active views

Reports

The Reports tab focuses on visual year-to-date analytics. You can drill into a single user or team.

Sub-tabs & Search

  • Users / Teams sub-tabs — choose what the search box autocompletes against.
  • Search box — type to filter; click a result to scope every chart and metric on the page to that user or team. A pill at the top shows the active filter; click the × to clear.

Summary Metrics

Four cards across the top — Activities (YTD), Points (YTD), Course Types, and Average Points per Activity — recompute when a filter is applied.

Charts

  • Monthly Performance (YTD) — bar chart of points earned per month for the current year.
  • Course Type Breakdown — doughnut chart of points by course type (top 8).

Exporting Data

There is no single global export button. Use Export Selected in the bulk-action toolbars of the Users and Activities tabs to download selected rows as JSON.

Backups: Because the only persistence layer is your browser's local storage, regularly export the selected rows you care about or copy the cloudCompData entry from DevTools → Application → Local Storage for a full snapshot.

Tips & Best Practices

Daily

  • Check the dashboard for a quick performance overview
  • Award manual points for special achievements promptly

Weekly

  • Import new activity data — the deduplicator skips rows you've already imported
  • Keep team memberships current

Monthly / Quarterly

  • Switch the dashboard time filter to Month or Quarter to share standings with the team
  • Run an end-of-period campaign so historical leaderboards stay scoped to the right window even as activities accumulate
  • Export selected rows from the Users tab as a snapshot of the rankings you announced

Troubleshooting

IssueSolution
Data not savingCheck browser settings → Privacy → ensure localStorage is enabled (Safari Private Browsing blocks it)
Import skips rows with warningsVerify the file has the required columns (Email, Course ID, Course Title, Status) and a valid Completed on date for completed rows
Last Activity shows — or 1/1/1970Old data from an .xlsx import that hit the Excel-serial-date bug. Re-import the original file with this version — cellDates:true now parses dates correctly
Points incorrect for new importsAdjust the Configuration tab values and click Save Changes; the new values apply to future imports/manual awards only
Teams not showing pointsMake sure members are assigned via the team's Manage option; points are derived from member activity totals
App slow after a big importImports persist in one batch from v1.3 on. If you upgraded from an earlier version, a one-time re-import resolves any leftover slowness from the previous schema

Important: Clearing your browser's site data permanently deletes everything CloudComp has stored. Before clearing, copy the cloudCompData entry from DevTools → Application → Local Storage to a safe place.
